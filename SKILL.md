# 任务耗时预估器

让用户在任务开始前就知道大概要等多久，合理安排自己的时间。

## 核心能力

1. **预估耗时**：拆解任务步骤，按基准耗时累加，输出一句自然的时间预估
2. **自动注入**：首次使用时检查用户的 AGENTS.md，如果没有预估规则，询问用户是否注入，注入后全局生效，不再需要手动触发本 Skill

## 使用流程

### Step 1：首次使用时自检 & 注入

检查用户的 `AGENTS.md`（路径：`~/.qoderwork/awareness/main/AGENTS.md`）中是否已包含「任务耗时预估」相关内容。

- **如果已有**：跳过，直接进入 Step 2
- **如果没有**：向用户说明这个 Skill 的作用，询问：

> 这个 Skill 可以在每次执行多步任务前，自动给你一个时间预估（比如「⏱ 预计5分钟，可以去倒杯水」）。
> 我可以把这个能力写进你的工作手册（AGENTS.md），这样以后每次对话都会自动生效，不需要再手动触发。要帮你加上吗？

用户同意后，将下方「注入内容」追加到 AGENTS.md 末尾。

### Step 2：执行预估

对当前任务执行预估流程：

1. **创建 TodoWrite 时，第一条固定为耗时预估**（如「⏱ 预计 X 分钟」），状态直接标 completed，确保预估不会被跳过
2. **拆步骤**：通过 TodoWrite 将任务分解为具体操作步骤
3. **累加耗时**：按下方基准耗时表，累加每一步的预计耗时
4. **乘缓冲系数**：总时间 × 1.5
5. **输出预估**：TodoWrite 第一条 + 回复第一句话同时体现

### Step 3：动态更新

执行过程中如发现实际复杂度超出预期（多次重试、意外依赖等），主动更新：
「比预期复杂一些，预计还需要多 X 分钟」

## 单步基准耗时参考

| 操作类型 | 基准耗时 | 说明 |
|---------|---------|------|
| 联网搜索 | ~30s/次 | WebSearch / WebFetch |
| 读文件 | ~10s/次 | Read / Glob / Grep |
| MCP 工具调用 | ~15s/次 | 钉钉、浏览器等外部工具 |
| 文件编辑 | ~15s/次 | Edit / Write |
| 文档生成 | ~2min/份 | 完整 md/docx/pptx/html |
| 代码编写 | ~2min/份 | 一个完整函数或模块 |
| Subagent (Task) | ~1-2min/个 | 并行可不叠加 |
| 浏览器自动化 | ~2min/页 | navigate + 交互 + 截图 |
| 数据分析脚本 | ~3min/轮 | 写脚本 + 执行 + 解读结果 |

## 输出格式

简洁自然地嵌在第一句话里，附上生活化建议：

- < 1min：不提示，直接做
- 1-3min：「⏱ 预计 X 分钟，马上好」
- 3-10min：「⏱ 预计 X 分钟，可以去倒杯水」
- 10-30min：「⏱ 预计 X 分钟，建议先去处理其他事」
- > 30min：「⏱ 预计超过半小时，建议去做别的事再回来看结果」

## Pitfalls

- 基准耗时是经验值，实际波动较大（网络延迟、模型响应速度等），只给区间不给精确数字
- 注入 AGENTS.md 前必须先读取当前内容，追加到末尾，不要覆盖用户已有的规则
- 如果用户的 AGENTS.md 已经有类似的预估规则（哪怕措辞不同），不要重复注入，告知用户已有即可
- 缓冲系数 1.5x 宁可高估也不要低估——用户提前完成比超时等待体验好得多

## Verification

- 检查 AGENTS.md 是否成功写入（Read 文件确认）
- 对一个示例任务执行预估，验证输出格式是否符合预期
